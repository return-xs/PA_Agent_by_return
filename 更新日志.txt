2.支持增量更新：
现在提交分析时会先找同品种/同周期的最近成功记录，计算当前快照相对它新增了多少根已收盘 K 线。新增数 <= 设置阈值 就走增量分析，超过阈值或找不到可靠上一轮记录就完整重跑。
设置位置：设置 -> 通用设置 -> 增量分析最大新增K线，默认 10。设为 0 可关闭增量分析。
流程现在是：
点击提交分析
  ↓
生成当前 K 线快照
  ↓
查找最近成功 record（同 symbol/timeframe）
  ↓
用上一轮最新 K 线在当前快照里的位置计算新增数量
  ↓
新增数量 <= 阈值
    → 增量 Stage1
    → Stage2 接着增量 Stage1 完整对话续写
  否则
    → 完整两阶段分析

3.实现三档交易倾向
档位含义
设置	含义
保守
与改版前一致（默认）
均衡
在同样决策树下，比保守更愿意下单
激进
三档中最积极，仍遵守 10.3、§14 等硬规则
实现方式
设置：设置 → 通用设置 → 交易倾向（下拉三档）
作用范围：只影响阶段二 prompt，阶段一闸门不变
注入位置：阶段二 user 提示最前面增加 ## 交易倾向 说明块
记录：RecordMeta.decision_stance 会保存当次档位，便于回放对比
状态栏：分析中会显示 倾向:保守/均衡/激进
三档差异（摘要）
保守：典型信号、R:R 倾向 ≥1.5:1、边际 10.3 倾向否、有疑虑就不下单（现状）。

均衡：允许次优 setup、约 1.2:1 可过 10.3、仅明确触犯 §14 才不下单、顺势/边界更倾向找单。

激进：更早/更不完美入场、约 1.0:1 在结构一致时可过 10.3、主动找可执行方案。

三档都保留：完整 decision_trace、真实 10.3 数值判断、§14 硬禁止，不会后处理把「不下单」改成下单。

使用说明
未改设置时默认 保守，行为与之前相同
想提高下单比例：先试 均衡，仍偏观望再试 激进
每次提交都会读取当前设置；改档位后需重新点「提交分析」或「增量分析」

4.增加胜率和盈亏比的展示：
 阶段二 JSON 新增字段（必填）
字段	说明
estimated_win_rate
0–100 整数，表示「按该 entry/stop/target 成交后获胜的概率」
estimated_win_rate_reasoning
简要说明依据（须与 §10.3 交易者方程一致）
规则：

有下单：estimated_win_rate 必填整数（取一个最可能值，例如 47，不要写 "45-50" 字符串）
不下单：estimated_win_rate 为 null
与 trade_confidence 区分：

trade_confidence：对「这笔决策是否值得做」的把握
estimated_win_rate：对「这笔单成交后赚钱概率」的判断（10.3 方程里用的胜率）
